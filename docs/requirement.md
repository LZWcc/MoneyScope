# MoneyScope 需求分析

## 一、项目背景

大学生和普通个人用户经常需要记录日常消费，但传统表格维护成本较高，也不方便做分类统计。MoneyScope 希望通过一个轻量的 Python 应用，帮助用户记录收入和支出，并通过图表查看消费结构和月度趋势。

## 二、项目目标

1. 完成一个可运行、可演示、可解释的 Python 课程项目。
2. 使用 Streamlit 实现网页式交互界面。
3. 使用 SQLite 保存收入、支出、分类和预算数据。
4. 使用 pandas 和 plotly 完成统计分析与可视化。
5. 满足课程对数据持久化、异常处理、面向对象、AI 日志、Git 协作和文档的要求。

## 三、用户角色

### 普通用户

普通用户可以添加收入和支出记录，查看消费明细和统计图表，并根据预算提醒调整消费习惯。

### 项目维护者

项目维护者负责维护数据库结构、业务逻辑、界面展示、测试记录和课程提交材料。

## 四、功能需求

### 1. 交易记录管理

- 添加收入记录。
- 添加支出记录。
- 查看全部交易记录。
- 删除错误记录。
- 按月份、类型和分类筛选记录。

### 2. 数据统计分析

- 统计指定月份的总收入、总支出和结余。
- 统计不同消费分类的支出金额。
- 展示月度收入、支出变化趋势。
- 展示分类消费占比。

### 3. 预算提醒

- 设置月度预算或分类预算。
- 当支出超过预算或接近预算时给出提示。

### 4. CSV 导入与导出

- 从 CSV 文件导入交易记录。
- 导出交易记录或统计结果。
- 对 CSV 字段缺失、金额格式错误等情况给出错误提示。

### 5. 文档与课程材料

- README 说明安装和运行方式。
- AI_LOG 记录至少 8 条有效 AI 使用过程。
- docs 记录需求、数据库设计和测试结果。
- reports 准备项目报告和 PPT 大纲。
- screenshots 保存运行截图。

## 五、非功能需求

1. 稳定性：常见错误应有清晰提示，不能直接崩溃。
2. 可维护性：数据库、模型、业务逻辑、图表和界面分文件维护。
3. 可解释性：核心函数和类需要 docstring，答辩时能解释关键代码。
4. 隐私性：不提交真实个人消费数据，仓库只保留示例数据。
5. 可演示性：项目应能在课堂演示环境中快速安装和运行。

## 六、MVP 范围

第一版必须完成：

1. SQLite 数据库初始化。
2. 添加收入和支出记录。
3. 查看和筛选交易记录。
4. 月度收支汇总。
5. 分类消费图表。
6. CSV 导入与导出。
7. 简单预算提醒。

暂不做：

1. 用户登录。
2. 云端同步。
3. 多人在线协作。
4. 复杂机器学习推荐。
5. 独立 Vue 前端。
