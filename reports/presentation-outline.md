# MoneyScope PPT 大纲

> 说明：本大纲为演示规划，每页含讲解要点。功能演示与截图需在 MVP 运行后由成员 C、E 补充真实画面。

## 1. 项目背景（1 页）

- 痛点：个人收支记录靠表格，维护成本高、难以分类统计与看趋势。
- 目标：用轻量 Python 应用完成记录、统计、可视化与预算提醒的完整闭环。

## 2. 技术选型（1 页）

- Python 3.10+ 为主语言。
- Streamlit 做网页交互界面，开发快、适合数据应用。
- SQLite 做本地持久化，免安装、便于课程演示。
- pandas 做统计分析，plotly 做可视化。
- 一句话理由：聚焦 Python 数据主线，避免引入重前端削弱评分重点。

## 3. 功能展示（2–3 页，配截图）

- 添加收入/支出记录，金额与日期校验。
- 按月份、类型、分类筛选交易明细，删除错误记录。
- 月度收入/支出/结余概览。
- 分类支出占比图与月度收支趋势图。
- 预算设置与接近/超预算提醒。
- CSV 导入示例数据与导出记录。

## 4. 系统结构（1 页）

- 分层架构图：UI → 服务层 → 模型/数据库层 → SQLite。
- 目录职责：app（代码）、data（示例数据）、docs（需求/设计/测试）、reports（报告/PPT）、tests（测试）。
- 强调：界面层不写 SQL，业务集中在服务层，便于维护与解释。

## 5. 数据库设计（1 页）

- 三张表：transactions、categories、budgets。
- 关键约束：金额 CHECK > 0、类型 CHECK in (income, expense)、唯一约束防重复。
- 设计取舍：分类先用文本存储，降低复杂度，后续可外键化。

## 6. 核心代码（1–2 页）

- 面向对象：Transaction / Budget 数据类与 __post_init__ 校验。
- 数据库初始化与参数化查询（防 SQL 注入）。
- 统计：pandas 分组汇总；预算判断（80% 接近、超额提醒）。
- 图表：plotly 生成饼图与趋势折线图。

## 10. 总结与改进（1 页）

- 亮点：完整收支闭环、清晰分层、中文异常提示、AI 开发记录。
- 不足：分类文本存储、无用户系统、统计维度有限。
- 改进：自动分类、数据备份恢复、更多可视化维度。
